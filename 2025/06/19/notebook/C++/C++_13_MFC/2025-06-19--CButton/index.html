MFC CButton 类 详解

下面对 MFC 中的 CButton 类做全面剖析，涵盖其定位、继承结构、样式与创建方式、核心成员函数、消息映射、以及典型用法示例和注意事项。

1. 类的定位与继承结构

  • 定位
    CButton 是 MFC 对 Win32 按钮控件（BUTTON 类）的封装，支持常见的按钮类型（普通按钮、复选框、单选按钮、图片按钮等）及其交互行为（点击、焦点、状态切换）。

  • 继承层次

2. 按钮样式与创建方式

2.1 样式（dwStyle）

样式常量 含义
BS_PUSHBUTTON 标准可按下按钮
BS_DEFPUSHBUTTON 默认按钮（回车触发）
BS_CHECKBOX 复选框
BS_AUTOCHECKBOX 复选框，点击自动切换
BS_RADIOBUTTON 单选按钮
BS_3STATE 三态复选框
BS_AUTO3STATE 三态复选框，点击自动循环
BS_GROUPBOX 分组框（不响应点击，只作视觉分隔）
BS_OWNERDRAW 自绘按钮，需要在 DrawItem 中实现绘制
BS_BITMAP 按钮显示位图
BS_ICON 按钮显示图标
BS_FLAT (Vista+) 扁平风格按钮

2.2 创建方式

  • 资源方式
    在对话框或对话框模板资源中直接拖放按钮控件，设置属性 ID、样式、标题等，MFC 会在 DoDataExchange 或 OnInitDialog 时自动 subclass。

  • 动态创建

3. 核心成员函数

函数 功能说明
BOOL Create(...); 动态创建按钮控件
void SetButtonStyle(UINT nStyle, BOOL bRedraw = TRUE); 修改按钮样式（如切换为复选框或单选按钮）
UINT GetButtonStyle() const; 获取当前按钮样式
void SetCheck(int nCheck); 设置复选框/单选按钮状态：BST_CHECKED, BST_UNCHECKED, BST_INDETERMINATE
int GetCheck() const; 查询当前状态
void SetIcon(HICON hIcon); 为按钮设置图标
void SetBitmap(HBITMAP hBmp); 为按钮设置位图
void DrawItem(LPDRAWITEMSTRUCT lpDrawItemStruct); (虚) 自绘按钮时覆写此函数，自行绘制外观
BOOL PreTranslateMessage(MSG* pMsg); 可在此处对快捷键（空格/回车）进行预处理

4. 消息映射与事件处理

MFC 通过消息映射宏将按钮点击消息映射到成员函数：

  • ON_BN_DOUBLECLICKED：双击消息，仅在 BS_NOTIFY 样式下有效
  • ON_CONTROL_RANGE：对一系列按钮统一处理（如单选按钮组）

7. 注意事项

  1. 控件子类化
    如果要在对话框类中直接操作 CButton 成员变量，需在 OnInitDialog 中调用 SubclassDlgItem 或使用 DDX_Control 映射。

  2. 资源与动态创建混用
    避免对同一个控件既在资源中存在又动态 Create，否则会出现句柄冲突。

  3. 图像资源管理
    使用 SetBitmap/SetIcon 后，需确保所传入的 GDI 对象在按钮析构前有效，并在不再使用时适当 DeleteObject 或 DestroyIcon。

  4. 无边框／透明按钮
    高级效果可结合 BS_FLAT、WS_EX_TRANSPARENT，再自绘或在父窗口背景上直接绘制，实现无边框或透明按钮。

通过对 CButton 的样式、创建、消息处理及自绘机制的深入掌握，你可以在 MFC 应用中创建多样化、高度定制化的按钮控件。
