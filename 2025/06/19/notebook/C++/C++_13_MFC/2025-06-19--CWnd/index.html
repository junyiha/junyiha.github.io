MFC CWnd 类 详解

下面对 MFC 中的 CWnd 类做全面剖析，涵盖其定位、继承结构、核心功能、关键成员与常用方法，以及使用示例和注意事项。

1. 类的定位与继承结构

  • 定位
    CWnd 是 MFC 中所有窗口类的基类，封装了对 Windows 原生窗口（HWND）的管理和操作逻辑。几乎所有可见的界面元素（框架窗口、对话框、控件、子窗口）都派生自它。

  • 继承层次

    • CObject 提供运行时类型信息和序列化支持
    • CCmdTarget 增加了消息映射与命令（菜单/工具栏）分发机制
    • CWnd 则直接对应一个 Windows 窗口句柄 m_hWnd

2. 核心功能

  1. 窗口创建与销毁

    • Create(...) / CreateEx(...)：动态创建窗口，封装了对 CreateWindowEx 的调用
    • DestroyWindow()：销毁与之关联的原生窗口

  2. 消息发送与分发

    • SendMessage() / PostMessage()：向自身或其他窗口发送/发布消息
    • PreTranslateMessage()：在消息进入消息泵前预处理（如快捷键）

  3. 绘图与更新

    • Invalidate() / InvalidateRect()：标记客户区无效，触发 WM_PAINT
    • UpdateWindow()：强制立即重绘

  4. 坐标与布局

    • MoveWindow() / SetWindowPos()：调整位置和大小
    • GetClientRect() / GetWindowRect()：获取客户区和窗口在屏幕坐标系中的矩形

  5. 子窗口管理

    • GetParent() / SetParent()：访问或修改父窗口
    • GetDlgItem()：在对话框或父窗口中查找子控件

  6. 资源与属性

    • SetWindowText() / GetWindowText()：设置或获取标题栏/控件文本
    • SetFont() / GetFont()：管理窗口或控件字体

  7. 对话框支持

    • DoModal()（对于 CDialog 派生类）
    • EndDialog()：结束模态对话框并返回结果

3. 关键成员变量

m_pParent CWnd* 指向父窗口（若有）

7. 注意事项

  1. 窗口句柄有效性：在调用任何操作前，确保 m_hWnd 已创建（如在 OnCreate 之后使用）。
  2. 线程限制：窗口应创建在其消息循环所在的 GUI 线程中，避免跨线程操作导致不可预期行为。
  3. 父子关系管理：销毁父窗口时，所有子窗口会被一起销毁；不要在子窗口析构时再手动销毁原生句柄。
  4. 资源清理：若手动注册了类（RegisterClass）或创建了 GDI 资源，应在合适时机释放。

8. 小结

  • CWnd 是 MFC 窗口体系的核心，提供了窗口生命周期管理、消息分发、绘图和布局等一系列基础能力。
  • 通过继承 CWnd 并结合消息映射宏，用户可快速定制自己的窗口或控件类。
  • 深入理解 CWnd 的方法和生命周期，有助于编写健壮、高效的 MFC 界面代码。

MFC CWnd::UpdateWindow() 函数 详解

CWnd::UpdateWindow() 是 MFC 对 Win32 API UpdateWindow 的封装，用来强制立即绘制窗口的客户区（client area），而不等待标准的消息循环触发 WM_PAINT。下面从函数原型、调用时机、内部机制、与 Invalidate 的配合，以及示例和注意事项等方面做详细讲解。

1. 函数原型

  • 返回值

    • 如果窗口有未处理的 WM_PAINT 消息并且已成功绘制，返回 非零。
    • 如果没有客户区需要重绘或出错，返回 零。

2. 与 Invalidate() 的区别

  • Invalidate() / InvalidateRect()

    • 标记客户区（或部分区域）为“脏”状态，告诉系统该区域需要重绘。
    • 并不立即绘制，只是向消息队列投递一个 WM_PAINT（或修改 UpdateRgn），由系统在空闲时统一处理。

  • UpdateWindow()

    • 检查窗口的“脏”区域：如果存在未处理的更新区域，直接调用 Win32 BeginPaint/EndPaint（或 SendMessage(WM_PAINT)) 来立即执行重绘。
    • 不会影响新的 Invalidate 调用，后续仍可继续标记和更新。

3. 内部工作流程

  1. 检查更新区域
    MFC 内部首先调用 Win32 的 GetUpdateRect 来判断该窗口是否存在未处理的绘制区域。

  2. 触发绘制

    • 如果有待绘制区域，则直接调用 SendMessage(m_hWnd, WM_PAINT, 0, 0)，
    • 或者调用底层的 DefWindowProc+BeginPaint/EndPaint，从而触发窗口过程中的 OnPaint() 回调。

  3. 清除脏区标记

    • 一旦绘制完成，系统会清除该区域的“脏”标记，保证下次绘制只发生在再次调用 Invalidate() 后。

5. 注意事项

  • 不要过度调用
    在大量重绘场景（如鼠标拖动、窗口移动）中频繁 UpdateWindow() 会导致性能下降，应该尽量让系统批量处理 WM_PAINT。

  • 非客户区不受影响
    UpdateWindow() 仅针对客户区。若要更新非客户区（标题栏、边框等），需使用 RedrawWindow(..., RDW_FRAME | RDW_INVALIDATE)。

  • 线程安全
    应在创建此窗口的 GUI 线程中调用，不要跨线程直接 UpdateWindow()，否则可能出现线程冲突或绘制失效。

6. 与 RedrawWindow 的对比

方法 作用 灵活度
UpdateWindow() 仅立即绘制客户区已标记的脏区 简单直接
RedrawWindow() 可指定客户区、非客户区，支持边框、内部子窗口 最大灵活（多标志组合控制）

若需要更精细地控制重绘（如同时重绘边框、所有子窗口），建议使用 CWnd::RedrawWindow。

小结

  • CWnd::UpdateWindow() 强制立即处理并绘制所有已标记的客户区脏区，跳过系统空闲时机。
  • 与 Invalidate() 配合可实现即时更新，常用于动画、进度反馈、首次渲染等场景。
  • 在高频率重绘场合，务必衡量 UpdateWindow 的调用频率，以免造成性能瓶颈。
