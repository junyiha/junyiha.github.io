MFC CFrameWnd 是什么

在 MFC（Microsoft Foundation Classes）中，CFrameWnd 是表示“框架窗口”（frame window）的基类，通常用来作为应用程序主窗口或文档/视图架构中每个视图的外壳窗口。它主要负责：

  1. 窗口外观与布局

    • 自动创建并管理标题栏（title bar）、菜单栏（menu bar）、工具栏（toolbar）、状态栏（status bar）等标准“窗口框架”元素。
    • 支持停靠（dockable）工具栏和可停靠的控件栏。

  2. 消息路由与命令分发

    • 在菜单、工具按钮或加速键（accelerator）触发命令时，将消息路由到合适的 CWinApp、文档（CDocument）或视图（CView）对象。
    • 提供 OnCreate()、OnSize()、OnDrawFrame() 等虚函数，方便自定义创建过程和响应大小调整。

  3. 文档/视图支持

    • 在单文档界面（SDI）应用中，主框架通常直接继承自 CFrameWnd。
    • 在多文档界面（MDI）应用中，则使用其派生类 CMDIFrameWnd 作为主框架，而每个子窗口框架继承自 CMDIChildWnd。

关键函数和成员

成员／函数 作用
PreCreateWindow 在窗口创建前修改 CREATESTRUCT（如样式、类名、窗口名等）
OnCreate 窗口创建时回调，常在此处创建和初始化工具栏、状态栏等控件
m_wndToolBar 内置的工具栏成员（如果你在 OnCreate 中创建了它）
m_wndStatusBar 内置的状态栏成员（在 OnCreate 中初始化）
LoadFrame(UINT id) 根据资源 ID 加载菜单、工具栏和加速表，并创建框架窗口
GetActiveDocument() 在文档/视图框架中，快速获取当前激活的文档对象

下面展示一个最基础的 SDI 应用中，如何用 CFrameWnd 创建主框架窗口并加载工具栏和状态栏。

  • **OnCreate**：窗口实际创建时调用，通常在这里实例化并配置工具栏、状态栏，以及设置停靠行为。

小结

  • CFrameWnd 是 MFC 应用的“窗口框架”基类，负责管理标题栏、菜单栏、工具栏、状态栏等 UI 构件，以及命令分发和消息路由。
  • 通过重载 PreCreateWindow、OnCreate 等关键虚函数，你可以灵活地定制主窗口的外观和行为。
  • 对于 SDI 应用，主框架直接继承自 CFrameWnd；对于 MDI 应用，则通常使用其派生类 CMDIFrameWnd／CMDIChildWnd。

理解并掌握 CFrameWnd 的使用，是构建功能丰富、界面规范的 MFC 桌面应用的基础。

MFC CFrameWnd::Create() 函数 详解

CFrameWnd::Create() 是 MFC 中用于动态创建框架窗口（frame window）的成员函数，它在运行时根据指定的窗口类和样式生成一个顶层窗口，常用于在不通过资源模板（.rc）而直接代码创建主窗口或子框架窗口。

2. 参数详解

  • lpszClassName

    • 要创建的窗口类名。若你使用默认的 AfxRegisterWndClass 注册的 MFC 类，可直接传入 AfxRegisterWndClass(0) 的返回值。
    • 对于自定义窗口类，需要先通过 WNDCLASS 或 AfxRegisterClass() 注册。

  • lpszWindowName

    • 窗口标题栏文字。可为 NULL 表示无标题。

  • dwStyle

    • 标准窗口风格位，如：

      • WS_OVERLAPPEDWINDOW（重叠窗口，带标题栏、边框、系统菜单、最大最小按钮）
      • 可按需添加 WS_VISIBLE（默认不可见时需手动 ShowWindow）

  • rect

    • 初始矩形，给出窗口左上角和右下角的坐标，基于父窗口客户区或屏幕（若无父窗口）。

  • pParentWnd

    • 父窗口指针。对于顶层框架通常传 NULL，对于子框架或控件则传相应指针。

  • lpszMenuName / nIDMenu

    • 菜单资源的字符串名称或整数 ID，指定菜单条；若无菜单可传 NULL 或 0。

  • dwExStyle

    • 扩展样式，如 WS_EX_APPWINDOW（确保在任务栏显示）或 WS_EX_CLIENTEDGE（带凹陷边框）。

  • pContext

    • 文档/视图上下文，仅在 SDI/MDI 应用通过 CSingleDocTemplate/CMultiDocTemplate 创建视图时使用，以便框架内部映射视图类和文档类。对手工创建的纯窗口框架通常传 NULL。

3. 返回值

  • 成功返回 TRUE
  • 失败返回 FALSE 并且可通过 GetLastError() 或在调试时查看 TRACE 输出获得原因（如类名未注册、样式冲突等）。

4. 内部流程

  1. PreCreateWindow 调用

    • 在调用底层 CreateWindowEx 之前，MFC 会先调用虚函数 PreCreateWindow(CREATESTRUCT& cs)。
    • 你可重载此函数修改 cs.style、cs.dwExStyle、cs.lpszClass、cs.lpszName 等，进而影响最终窗口属性。

  2. 调用 Win32 API

    • CreateWindowEx(cs.dwExStyle, cs.lpszClass, cs.lpszName, cs.style, …, hInstance, this)。
    • this 指针通过 CREATESTRUCT.lpCreateParams 传递给窗口过程，以便在 WM_NCCREATE/WM_CREATE 时绑定 CWnd 对象和 HWND。

  3. 关联 CWnd 对象

    • 创建成功后，MFC 在内部将生成的 HWND 与当前 CFrameWnd 对象关联，并初始化 m_hWnd、m_pWndNext 等链表。

  4. OnCreate 回调

    • Win32 发送 WM_CREATE 消息，触发 CFrameWnd::OnCreate，你可在此处创建工具栏、状态栏、停靠窗口等子控件。

6. 注意事项

  • 调用时机

    • 必须在 CWinApp::InitInstance 或窗口对象已构造后调用，且确保窗口类已注册（如放在 PreCreateWindow 中或显式 AfxRegisterWndClass）。

  • 资源加载

    • 若希望 MFC 自动加载工具栏、加速器等，请使用 LoadFrame 而非 Create；Create 仅生成空框架窗口。

  • 文档/视图应用

    • 在 SDI/MDI 应用中，不推荐手工调用 Create，而应依赖框架通过模板自动创建，以保证视图正确挂载。

通过重载 PreCreateWindow、合理传递参数，并在 OnCreate 中补充子控件初始化，CFrameWnd::Create() 能让你以最细粒度的方式动态构建自定义窗口框架。
