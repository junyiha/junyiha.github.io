cv::dnn::Net::setPreferableTarget() 函数 详解

cv::dnn::Net::setPreferableTarget(targetId) 用来指定在所选后端（backend）中，将网络推理调度到哪个设备或硬件加速路径。它的原型为：

一、常见 targetId 枚举

枚举值 含义
cv::dnn::DNN_TARGET_CPU 在 CPU 上执行（默认）
cv::dnn::DNN_TARGET_OPENCL 使用 OpenCL 加速，设备由 OpenCL 后端选择（GPU/加速器）
cv::dnn::DNN_TARGET_OPENCL_FP16 OpenCL 半精度（FP16）加速，仅当设备与驱动支持时有效
cv::dnn::DNN_TARGET_MYRIAD Intel Movidius NCS（VPU）设备
cv::dnn::DNN_TARGET_CUDA NVIDIA GPU（CUDA）
cv::dnn::DNN_TARGET_CUDA_FP16 NVIDIA GPU 半精度（FP16）
cv::dnn::DNN_TARGET_CUDA_FP16_TENSORRT NVIDIA TensorRT（FP16）加速
cv::dnn::DNN_TARGET_HDDL Intel Vision Accelerator Design (HDDL)
cv::dnn::DNN_TARGET_VULKAN Vulkan Compute 加速
cv::dnn::DNN_TARGET_FPGA FPGA（需专门编译并支持）

提示：并非所有编译版本都支持上述所有 target；可用的 target 取决于你的 OpenCV 编译时配置和运行时环境。

二、与 setPreferableBackend() 的配合

  • Backend 决定用哪套“引擎”来驱动推理（OpenCV、CUDA、Vulkan、Inference Engine…）。
  • Target 决定在该引擎内跑在哪个具体设备上（CPU、GPU、VPU、半精度、TensorRT…）。

示例组合：

如果只设置 target 而不设置 backend，OpenCV 会先用默认 backend（通常是 OPENCV），再用给定 target。但最佳实践是两者都指定。

三、使用示例

四、注意事项

  1. 编译依赖

    • 要使用 CUDA 或 TensorRT target，需在 OpenCV 编译时开启对应模块（如 -DWITH_CUDA=ON、-DWITH_TENSORRT=ON）。
    • OpenCL target 需正确安装 ICD 驱动，并在 CMake 中启用 -DWITH_OPENCL=ON（大多数预编译包已开启）。

  2. 性能对比

    • 不同 target 在启动延迟、吞吐量、精度（FP32/FP16/INT8）上差异显著，建议在目标平台做基准测试选最优组合。

  3. 模型兼容

    • 某些后端/target 对模型层支持不全，可能出现“不支持的 layer”或行为差异；可通过 OpenCV 日志级别或 try-catch 捕获并回退。

  4. 多设备场景

    • 如果系统中有多张 GPU 或多种加速设备，可通过 OpenCL API（先枚举）或 CUDA API 显式选择设备，然后再让 OpenCV DNN 走对应 target。

  5. 半精度与数值稳定

    • FP16 虽能提升吞吐，但可能引入精度波动；对关键场景应验证输出准确性。

通过合理地设置 setPreferableTarget，你可以在多种硬件与精度选项间灵活切换，从而在性能和资源约束间找到最佳平衡。

cv::dnn::Net::setPreferableBackend() 函数 详解

cv::dnn::Net::setPreferableBackend(backendId) 用于指定 DNN 模块在执行前向推理时所使用的“后端”（backend），即选用哪一套硬件加速或纯 CPU 实现来计算网络。它的原型及常用参数如下：

一、常见 backendId 枚举

枚举值 含义 备注
cv::dnn::DNN_BACKEND_DEFAULT 默认后端 由 OpenCV 决定（通常等同于 OPENCV+CPU）
cv::dnn::DNN_BACKEND_OPENCV OpenCV 自带实现（CPU） 纯 CPU、无额外依赖
cv::dnn::DNN_BACKEND_CUDA NVIDIA CUDA 后端 需要编译时启用 WITH_CUDA，并有 CUDA 驱动
cv::dnn::DNN_BACKEND_HALIDE Halide 框架加速 需要 OpenCV 编译时启用了 Halide 支持
cv::dnn::DNN_BACKEND_INFERENCE_ENGINE Intel OpenVINO™（IE）后端 需要编译时启用了 Inference Engine
cv::dnn::DNN_BACKEND_VKCOM Vulkan Compute 后端 需要 Vulkan SDK 支持
cv::dnn::DNN_BACKEND_TIMVX Arm® NN 的 TIM-VX 后端 用于嵌入式平台

注意：部分后端（CUDA/IE/Halide/VKCOM/TIMVX）并不是在所有预编译 OpenCV 包中都可用，需自行编译并启用对应选项。

二、与 setPreferableTarget() 的配合

后端决定了“使用哪种计算引擎”，而 target 决定了“在该后端中跑在哪个设备”。常见 target 枚举：

  • DNN_TARGET_CPU
  • DNN_TARGET_OPENCL / DNN_TARGET_OPENCL_FP16
  • DNN_TARGET_CUDA / DNN_TARGET_CUDA_FP16
  • DNN_TARGET_MYRIAD（Movidius NCS）
  • DNN_TARGET_FPGA 等

若只调用 setPreferableBackend 而不设 target，OpenCV 会为你选择默认 target（通常是 CPU）。

三、使用示例

四、注意事项

  1. 编译需求：要使用非 OPENCV 后端，需在编译 OpenCV 时开启相应模块（例如 -DWITH_CUDA=ON）。
  2. 性能对比：不同后端在加载时间、内存占用和吞吐量上表现各异，建议在目标平台上做基准测试。
  3. 模型兼容：部分后端（如 Inference Engine、Vulkan）对 Layer 支持有差异，可能出现“不支持层”错误，需要通过 plugin 或手动实现。

通过合理地选用 setPreferableBackend 与 setPreferableTarget，即可在 OpenCV DNN 流水线上灵活切换到最优的硬件加速路径，大幅提升推理性能。

C++ OpenCV cv::addWeighted() 详解

cv::addWeighted() 是 OpenCV 中一个非常常用的函数，主要用于图像加权融合（图像叠加），可以将两张图像按照一定的比例融合在一起，非常适合实现图像透明叠加、水印、特效处理等功能。

🧠 函数原型

📘 数学公式

dst(i,j) = saturate( src1(i,j)*alpha + src2(i,j)*beta + gamma )

  • alpha：控制第一张图像的权重
  • beta：控制第二张图像的权重
  • gamma：对融合结果增加亮度（可为 0）
  • saturate()：保证结果在合法图像范围（例如 uint8 时是 0～255）

✅ 参数详解

参数 类型 说明
src1 cv::Mat 输入图像1（与 src2 尺寸和类型必须相同）
alpha double 图像1的权重
src2 cv::Mat 输入图像2
beta double 图像2的权重
gamma double 结果图像的偏移值（可以理解为亮度调整）
dst cv::Mat 输出图像
dtype int 输出图像的数据类型，默认 -1 表示与输入相同

🎯 使用示例：图像融合

🧪 应用场景

  1. 添加透明水印：

    • 通过调整水印图的 alpha，可以实现透明叠加。

  2. 图像对比融合：

    • 用于多模态图像融合（如红外 + 可见光）。

  3. 图像过渡动画：

    • 在两张图之间逐步调整 alpha 和 beta 实现渐变效果。

🧩 注意事项

  • src1 和 src2 必须大小、通道数、类型一致，常见为 CV_8UC3。
  • 权重 alpha + beta 不一定必须等于 1，但这样做可以更容易理解。
  • 融合结果会自动进行饱和截断处理（避免像素值溢出）。
